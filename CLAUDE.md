## docs/ 폴더 규칙

- `docs/`는 현재 합의된 시스템 규칙과 구현 기준을 기록하는 곳입니다.
- 현재 동작, 구현 우선순위, 저장 구조, UI 규칙은 여기에 맞춰 정리합니다.
- 설계 아이디어가 실제 기준으로 승격되면 `docs/` 쪽 문서를 먼저 갱신합니다.

## docs/design/ 폴더 규칙

- `docs/design/`은 설계 아이디어, 백로그, 후보 규칙을 모아두는 참고 폴더입니다.
- 읽기는 가능하지만 코드 수정의 직접 근거로 사용하지 마세요.
- 명시적으로 "이 문서 기반으로 구현해줘" 지시가 있을 때만 참고하세요.
- `docs/design/`와 `docs/` 내용이 충돌하면 항상 `docs/`를 최신 기준으로 봅니다.
