# 기억 시스템

현재 결론: 지금 구현된 `기억`은 스마트폰 옆 패널에서 보는 행동 로그다. 중요한 행동과 결과를 카드처럼 기록하며, 대부분은 현재 맥락에 맞춰 유지되고 일부 항목만 더 오래 남길 수 있게 구조를 준비해 둔 상태다.

## 목적

- 플레이어가 방금 한 행동과 결과를 대사창 밖에서도 다시 확인할 수 있게 한다.
- 이동, 공고 지원, 금융 행동 같은 흐름이 한눈에 남도록 한다.
- 장기 기억 시스템으로 확장할 수 있도록 기본 데이터 구조와 UI 위치를 먼저 고정한다.

## 현재 확정 규칙

1. `기억` 버튼은 스마트폰과 분리된 별도 버튼으로 화면 우측에 고정된다.
2. 버튼을 누르면 별도 `기억 패널`이 열리고, 다시 누르거나 닫기 버튼으로 닫는다.
3. 각 항목은 `type`, `title`, `body`, `source`, `timestamp`, `tags`를 가진다.
4. 기본 보존 방식은 `volatile`이며, 현재 맥락이 바뀌면 오래된 항목이 정리된다.
5. `retention = ending` 항목만 맥락 변경 이후에도 남길 수 있다.
6. 기억 목록은 최신순으로 최대 64개까지만 유지한다.
7. 패널 열림 상태와 항목 목록은 `state.memory`에 저장된다.
8. 현재 단계에서 기억의 주 역할은 장기 버프가 아니라 `행동 로그`와 `상황 기록`이다.

## 현재 UI

### 위치

- 스마트폰 본체와는 별도인 우측 `기억` 버튼으로 연다.
- 패널은 스마트폰 패널 왼쪽 영역에 떠서 표시된다.

### 동작

1. 행동이나 결과가 발생하면 공통 기록 함수가 항목을 추가한다.
2. 플레이어가 `기억` 버튼을 누르면 패널이 열린다.
3. 패널 안에서 최신 항목부터 카드 목록을 본다.
4. 각 카드는 시간, 제목, 설명, 출처, 태그를 보여준다.

### 표시 원칙

- 현재는 미획득 항목을 보여주지 않는다.
- 숫자형 효과 문구보다 행동 자체의 기록을 우선 보여준다.
- 패널이 비어 있으면 안내 문구를 보여준다.

## 현재 기록되는 예시

- 방청소 시작 / 완료
- 오늘 공고 확인
- 예약 근무 시작 / 놓침
- 집 밖으로 나감 / 귀가 / 버스 이동
- 은행 입금 / 출금 / 송금
- 증시 결과 정리

## 현재 데이터 구조

```js
{
  panelOpen: false,
  entries: [
    {
      id: "memory-...",
      type: "action",
      title: "집 밖으로 나갔다",
      body: "골목 쪽으로 문을 열고 나갔다.",
      source: "집",
      timestamp: "DAY 01 · 08:30",
      day: 1,
      tags: ["이동", "외출"],
      retention: "volatile",
      contextKey: "1:outside:apt-front:home"
    }
  ]
}
```

## 구현 메모

- `js/systems/memory/memory-service.js`가 기억 상태 동기화, pruning, 추가를 담당한다.
- `recordActionMemory()`는 일반 행동 로그를 만드는 공통 진입점이다.
- 일반 항목은 `volatile`, 더 오래 남겨야 하는 항목만 `ending`으로 저장할 수 있다.
- 현재 문서의 장기 메타 기억 설계는 구현 대상이 아니라 `docs/design/memory-system.md`에서 별도로 관리한다.

## 아직 미확정인 확장

- 기억을 면접 확률이나 이벤트 조건에 직접 연결할지
- `ending` 보존 대상을 어떤 기준으로 남길지
- 로그형 기억과 장기 메타 기억을 언제 합칠지
- 태그 기반 분기 시스템을 현재 구조에 어떻게 얹을지
